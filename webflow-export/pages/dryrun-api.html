DryRun Simple API

The DryRun Simple API provides programmatic access to your organization's security data - findings, scans, deepscans, configurations, repositories, and insights.

  • Swagger UI: https://simple-api.dryrun.security/api-docs/index.html
  • OpenAPI (v3.0) spec: https://simple-api.dryrun.security/api-docs/v1/swagger.yaml
  • Base URL: https://simple-api.dryrun.security/v1

Authentication

For information on creating and managing API keys, see the API Access Keys page. All API requests require a valid API key sent in the Authorization header using the Bearer scheme.

Quick Start

Most endpoints are scoped to an account. You'll need:

  • account_id - provided by the DryRun Security platform (e.g., 12345678-1234-1234-1234-1234567890ab)
  • repository_id - a UUID for a repository

Typical workflow:

  1. List your accessible accounts.
  2. Pick an account, then list repositories in that account.
  3. Use repository IDs to fetch scans and findings.
curl   -H "Authorization: Bearer $DRYRUN_API_KEY"   https://simple-api.dryrun.security/v1/accounts

Endpoint Reference

MethodEndpointCategoryDescription
GET/v1/accountsAccountsList all accounts accessible by the API key.
GET/v1/accounts/{account_id}/repositoriesRepositoriesList all repositories for an account.
GET/v1/accounts/{account_id}/repositories/{repository_id}/scansScansList PR scans for a repository. Supports filtering by status, date, and pagination.
GET/v1/accounts/{account_id}/repositories/{repository_id}/scans/{id}ScansGet detailed PR scan results including findings.
GET/v1/accounts/{account_id}/repositories/{repository_id}/findingsFindingsList all PR findings for a repository. Each finding includes a dashboard_url.
GET/v1/accounts/{account_id}/all_findingsFindingsList all findings (PR and DeepScan) for an account. Supports filtering by days (required), finding_type, severity, repository_id, page, per_page.
POST/v1/accounts/{account_id}/findings/{finding_id}/triageFinding TriageSet a triage category for a finding. Request body: triage_category, one of: false_positive, wont_fix, accepted_risk, in_progress.
GET/v1/accounts/{account_id}/findings/{finding_id}/triageFinding TriageGet the triage status for a finding.
DELETE/v1/accounts/{account_id}/findings/{finding_id}/triageFinding TriageRemove the triage category from a finding (resets triage status).
GET/v1/accounts/{account_id}/deepscansDeepscansList all deepscans for an account.
GET/v1/accounts/{account_id}/repositories/{repository_id}/deepscansDeepscansList deepscans for a repository.
GET/v1/accounts/{account_id}/repositories/{repository_id}/deepscans/{deepscan_id}/resultsDeepscansList findings for a specific deepscan.
GET/v1/accounts/{account_id}/configurationsConfigurationsList configurations for an account.
POST/v1/accounts/{account_id}/configurationsConfigurationsCreate a new configuration.
PUT/v1/accounts/{account_id}/configurations/{id}ConfigurationsUpdate a configuration.
DELETE/v1/accounts/{account_id}/configurations/{id}ConfigurationsDelete a configuration.
GET/v1/accounts/{account_id}/analyzersAnalyzersList available analyzers. Use the slug field as the key in configuration analyzer settings.
GET/v1/accounts/{account_id}/custom_policiesCustom PoliciesList all Custom Code Policies for an account.
GET/v1/accounts/{account_id}/insightsInsightsRetrieve the daily insights digest. Supports a date query parameter (YYYY-MM-DD).

Conventions

  • IDs and scoping: account_id is required for most endpoints. repository_id is required for repository-scoped endpoints.
  • Response shape: Most list endpoints return a top-level data array.
  • Errors: If an item is not found, endpoints return 404 with {"error": "not found"}.

Support

If you have questions about authentication, account access, or expected responses, contact DryRun Security support and include the endpoint URL you called, the HTTP status code, and the request_id header (if present).
