Overview

DryRun Security uses a two-role model, Admin and Developer, that maps to your existing SCM platform roles. An account administrator can also manually promote any developer to Admin within the DryRun Security UI, regardless of their role in GitHub or GitLab.

DryRun Roles

Two roles exist in DryRun Security: Admin and Developer. Each role controls what a user can see and configure across the platform.

Admin

Full access to the platform across all repositories the customer has installed:

  • Risk Register (all repos)
  • Repositories page (all repos)
  • Pull Requests page (all PRs across all repos)
  • DeepScan page - view all DeepScan runs and reports, trigger new DeepScans
  • Code Policies - view and configure custom code policies
  • Insights page (including AI chat assistant and Daily Digest)
  • Configurations - edit all PR scanner configurations
  • Integrations - set up Slack, webhook integrations, and AI coding integrations (MCP/IDE)
  • Access Keys - generate API access keys
  • Install/uninstall repositories

Developer

Scoped access based on their repository membership in the SCM platform:

  • Risk Register - findings only for repos they have membership permission over
  • Repositories page - only repos they have membership permission over
  • Pull Requests - only PRs and findings for repos they have membership permission over
  • DeepScan - view only (cannot trigger new DeepScans)
  • Code Policies - view only

Cannot access: Insights page, Configurations, Access Keys, Integrations, or install/uninstall repos.

SCM Role Mapping

DryRun Security automatically maps SCM roles to DryRun roles at login. No manual configuration is required.

Admin Override

An account administrator can promote any developer to Admin within the DryRun Security UI. This override applies only within DryRun Security and does not change the user's role in GitHub or GitLab. To request an override, contact your account administrator or reach out to hi@dryrun.security.
