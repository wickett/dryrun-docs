Overview

DryRun Security uses a two-role model, Admin and Developer, that maps to your existing SCM platform roles. An account administrator can also manually promote any developer to Admin within the DryRun Security UI, regardless of their role in GitHub or GitLab.

SCM Role Mapping

DryRun Security automatically maps SCM roles to DryRun roles at login. No manual configuration is required.

SCM PlatformSCM RoleDryRun Role

Admin Override

An account administrator can request that DryRun Security promote a developer to Admin within the DryRun UI. This setting is managed by DryRun and is not self-serve. The override applies only within DryRun Security and does not change the user's role in GitHub or GitLab. To request an override, contact DryRun Security at hi@dryrun.security.
