Overview

DryRun Security uses a two-role model, Admin and Developer, that maps to your existing SCM platform roles. An account administrator can also manually promote any developer to Admin within the DryRun Security UI, regardless of their role in GitHub or GitLab.

SCM Role Mapping

DryRun Security automatically maps SCM roles to DryRun roles at login. No manual configuration is required.

Admin Override

An account administrator can promote any developer to Admin within the DryRun Security UI. This override applies only within DryRun Security and does not change the user's role in GitHub or GitLab. To request an override, contact your account administrator or reach out to hi@dryrun.security.
