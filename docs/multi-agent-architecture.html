Multi-Agent Architecture

How DryRun Security's agentic system coordinates specialized agents for comprehensive code security analysis.

Why Multi-Agent

DryRun Security uses a multi-agent architecture rather than a single monolithic model. Each agent specializes in a specific aspect of security analysis, and agents collaborate to produce comprehensive findings. This design mirrors how a senior security team works - one person focuses on authentication, another on injection, another on infrastructure - and the team's combined expertise exceeds what any individual could achieve.

The alternative - running all security analysis through a single model or pipeline - limits depth. A monolithic system must balance breadth against the context window and reasoning capacity available for any single vulnerability class. Multi-agent design removes this constraint.

Primary Agents

Code Review Agent

The Code Review Agent is the primary agent that runs on every pull request. It coordinates the overall review process: retrieving the PR diff and surrounding context, dispatching specialized sub-agents, collecting and validating findings, and posting the consolidated result as a PR comment and check.

The Code Review Agent also runs DryRun's Core Code Policies - a set of built-in checks covering the OWASP Top 10 and common vulnerability patterns. These are always active and require no configuration.

Custom Policy Agent

The Custom Policy Agent enforces your organization's Natural Language Code Policies alongside the standard SAST checks. When you write a policy like "no new API endpoints without authorization enforcement," the Custom Policy Agent interprets that policy and evaluates the PR against it, regardless of language or framework.

The Custom Policy Agent can call specialized sub-agents for SCA lookups, codebase analysis, and just-in-time research to fully evaluate whether a policy is satisfied.

DeepScan Agent

The DeepScan Agent performs full-repository security analysis on demand. While the Code Review Agent operates on PR diffs, the DeepScan Agent analyzes the entire repository tree at a given commit. This enables it to find vulnerabilities that span many files, were introduced gradually across multiple commits, or existed before DryRun Security was installed.

Codebase Insight Agent

The Codebase Insight Agent powers Code Insights - natural language queries about your codebase's security posture. Instead of navigating dashboards and exports, you ask questions like "what new API endpoints were added this week?" or "show me a chart of risky alerts by repo" and get precise, contextual answers.

Specialized Sub-Agents

Primary agents dispatch specialized sub-agents to handle specific vulnerability classes. Each sub-agent focuses on one area and uses Contextual Security Analysis for high-signal, low-noise findings:

  • SQLi Analyzer - SQL injection across languages, tracing sources to sinks
  • XSS Analyzer - cross-site scripting including framework-specific templating pitfalls
  • IDOR Analyzer - broken object-level authorization via data path tracing
  • Mass Assignment Analyzer - unsafe model binding from user input
  • SSRF Analyzer - server-side request forgery via user-controlled URLs
  • Secrets Analyzer - credentials in code, distinguished from test fixtures
  • General Security Analyzer - auth gaps, risky crypto, debug artifacts, deserialization, and more

See Security Analyzers for detailed documentation on each sub-agent.

How Agents Coordinate

When a PR is opened or updated, the flow works as follows:

  1. The Code Review Agent receives the PR event and retrieves the diff plus relevant surrounding files for context
  2. The agent dispatches specialized sub-agents in parallel, each analyzing the changes for their specific vulnerability class
  3. Sub-agents may call just-in-time research to look up framework behavior, CVE details, or license information
  4. The Custom Policy Agent evaluates any configured Natural Language Code Policies against the changes
  5. Findings from all agents are collected, validated for exploitability, and deduplicated
  6. The consolidated result is posted as a PR comment with findings, code references, and remediation guidance

Model-Independent Verification

Agents cross-check each other's findings to reduce the risk of any single model hallucinating or producing inconsistent results. This multi-agent verification is a key reason DryRun Security achieves high consistency and reliability in production environments. See Model-Independent Verification for more detail.
